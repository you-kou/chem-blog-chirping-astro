Cet article est la **page de test typographique** du thème. Si vous
modifiez les polices, la taille de la prose ou tout élément de la
pile typographique `prose-chirpy`, ouvrez cette page en mode clair et
sombre et vérifiez que tout reste équilibré.

La police de corps est **Source Sans 3**, la police de code est
**JetBrains Mono**, et la colonne de prose est plafonnée à
`--width-prose` (50 rem) pour que la longueur de ligne reste
confortable.

## Paragraphes et formatage en ligne

Vous pouvez écrire en **gras**, _italique_, ~~barré~~, `code en ligne`,
ou imbriqué **_gras italique_**. Les URL longues deviennent
cliquables : visitez [astro.build](https://astro.build). Les liens
externes s'ouvrent dans un nouvel onglet avec `rel="nofollow noopener"`
grâce à `rehype-external-links`.

Utilisez la balise `<kbd>` pour les raccourcis clavier : appuyez sur

<kbd>Cmd</kbd> + <kbd>K</kbd> ou <kbd>/</kbd> pour ouvrir la modale de recherche.

> « La typographie est l'art de donner au langage humain une forme
> visuelle durable. » — _Robert Bringhurst_

Un deuxième paragraphe maintient le rythme. Hauteur de ligne, marge
verticale et largeur maximale sont calibrées pour une lecture
confortable sur mobile et desktop.

## Titres

Les titres sont auto-slugifiés par `rehype-slug` et reçoivent une ancre
au survol par `rehype-autolink-headings` — survolez la gauche d'un
titre pour révéler le lien `#`.

### Titre de niveau 3

La table des matières flottante de droite affiche les H3 uniquement
quand vous êtes dans la section H2 parente.

#### Titre de niveau 4

Les H4 sont supportés et stylés, mais denses — à utiliser seulement
quand c'est vraiment nécessaire.

## Listes

Liste non ordonnée :

- Un premier élément avec un [lien vers l'accueil](/fr/).
- Un deuxième élément avec du **texte en gras** et `du code`.
- Un élément imbriqué :
  - Avec ses propres enfants.
  - Et un élément encore plus profond.
    - Trois niveaux, sans souci.

Liste ordonnée :

1. Choisir un sujet.
2. Faire le plan — H2 pour les sections, H3 pour les sous-sections.
3. Écrire un premier jet d'un trait, sans rien éditer.
4. Dormir dessus.
5. Éditer sans pitié.

## Code en ligne et raccourcis clavier

Utilisez `du code en ligne` pour les identifiants
(`SITE.showFeaturedImages`) ou les snippets courts (`bun astro dev`).
Dans les tableaux et les listes il reste lisible grâce à un fond
légèrement atténué.

## Blocs de code

Un exemple court — l'[article dédié aux blocs de code](/fr/posts/code-blocks-and-syntax-highlighting)
détaille toutes les options Expressive Code.

```ts title="src/utils/posts.ts" ins={5-8}
import { getCollection, type CollectionEntry } from 'astro:content';
import { SITE, type Locale } from '../config';

export function shouldShowHero(post: CollectionEntry<'posts'>): boolean {
  if (!post.data.heroImage) return false;
  return post.data.showFeaturedImage ?? SITE.showFeaturedImages;
}
```

## Callouts

Le thème embarque un composant MDX `<Callout>` à quatre variantes
sémantiques. Elles héritent des couleurs sémantiques de daisyUI, donc
restent lisibles dans les deux thèmes.

<Callout type="info" title="Info">
  À utiliser pour un contexte neutre que le lecteur doit remarquer sans agir.
</Callout>

<Callout type="success" title="Réussi">
  Vous venez de publier un article propre. Offrez-vous un thé.
</Callout>

<Callout type="warning" title="Attention">
  Évitez de définir `lang:` dans le frontmatter sauf nécessité absolue — le thème l'infère depuis le
  chemin du fichier.
</Callout>

<Callout type="error" title="Erreur">
  Oublier `pubDate` fait échouer le build avec une erreur Zod claire.
</Callout>

En Markdown pur, vous pouvez aussi utiliser la classe CSS bas-niveau :

<div class="callout callout-info">
  <strong>Note :</strong> mettez <code>toc: false</code> dans le frontmatter pour masquer la table
  des matières par article.
</div>

## Images

Les images Markdown en ligne occupent toute la largeur de la prose :

<Image
  src={imgDesk}
  alt="Lumière entrant par une fenêtre sur un bureau en bois"
  widths={[400, 800, 1200]}
/>

Une figure légendée avec `<figcaption>` :

<figure>
  <Image src={imgKeyboard} alt="Gros plan sur un clavier de MacBook" widths={[400, 800, 1200]} />
  <figcaption>Photo de Ales Nesetril sur Unsplash</figcaption>
</figure>

## Filets horizontaux

À utiliser avec parcimonie pour marquer un changement de sujet fort :

---

Après un filet horizontal, ce qui suit (texte, code, images) reçoit
une marge supérieure fraîche pour repartir nettement.
