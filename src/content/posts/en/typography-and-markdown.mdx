This post is the **typography test page** for the theme. If you change
fonts, prose sizing, or anything in the `prose-chirpy` typography stack,
load this page in both light and dark mode and confirm everything still
looks balanced.

The theme's body font is **Source Sans 3**, the code font is
**JetBrains Mono**, and the prose column is capped at `--width-prose`
(50 rem) so line lengths stay comfortable.

## Paragraphs and inline formatting

You can write **bold**, _italic_, ~~strikethrough~~, `inline code`, and
nested **_bold italic_**. Long URLs become clickable: visit
[astro.build](https://astro.build). External links open in a new tab
with `rel="nofollow noopener"` thanks to `rehype-external-links`.

Use the `<kbd>` tag for keyboard shortcuts: press <kbd>Cmd</kbd> + <kbd>K</kbd>
or <kbd>/</kbd> to open the search modal.

> "Typography is the craft of endowing human language with a durable
> visual form." — _Robert Bringhurst_

A second paragraph keeps the rhythm. Line height, vertical margin and
maximum width are tuned for comfortable reading on both phone and
desktop.

## Headings

Headings are auto-slugged by `rehype-slug` and given hover anchors by
`rehype-autolink-headings` — hover to the left of any heading on this
page to reveal the `#` link.

### Third-level heading

The right-hand TOC reveals H3s only while you are scrolled inside the
parent H2 section.

#### Fourth-level heading

H4s are supported and styled, but they are dense — reach for them only
when really needed.

## Inline code, code spans and keyboard shortcuts

Use `inline code` for identifiers (`SITE.showFeaturedImages`) or short
snippets (`bun astro dev`). Within tables and lists it stays readable
thanks to a slightly muted background.

## Fenced code blocks

A short example — the dedicated
[code blocks post](/posts/code-blocks-and-syntax-highlighting) covers
every Expressive Code option in detail.

```ts title="src/utils/posts.ts" ins={5-8}
import { getCollection, type CollectionEntry } from 'astro:content';
import { SITE, type Locale } from '../config';

export function shouldShowHero(post: CollectionEntry<'posts'>): boolean {
  if (!post.data.heroImage) return false;
  return post.data.showFeaturedImage ?? SITE.showFeaturedImages;
}
```

## Callouts

The theme ships an MDX `<Callout>` component with four semantic
variants. They inherit daisyUI semantic colours, so they stay legible
in both themes.

<Callout type="info" title="Info">
  Use this for neutral context that the reader should notice but not act on.
</Callout>

<Callout type="success" title="Success">
  You just shipped a clean post. Treat yourself to a tea.
</Callout>

<Callout type="warning" title="Warning">
  Avoid setting `lang:` in frontmatter unless you really need to — the theme infers it from the file
  path.
</Callout>

<Callout type="error" title="Error">
  Forgetting `pubDate` will fail the build with a clear Zod validation error.
</Callout>

In plain Markdown, you can also reach for the lower-level CSS class:

<div class="callout callout-info">
  <strong>Note:</strong> set <code>toc: false</code> in frontmatter to hide the table of contents on
  a per-post basis.
</div>

## Math (inline)

For full LaTeX math you must opt in with `math: true` (see the
[math post](/posts/latex-math-with-katex)). This page intentionally
omits that flag so we can verify that without it, KaTeX styles do
**not** load.
